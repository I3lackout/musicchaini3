# Vision

## One sentence

**MusicChain i3 turns listening and creation into a transparent, auditable music economy — not an opaque algorithm.**

## Problem

Music platforms concentrate value in intermediaries. Creators see opaque payouts. Listeners have no stake. “Crypto music” projects often ship a token first and a product second.

## Our direction

Build a **working streaming product** where:

1. **Listening and interaction** produce defined resource flows (McCC, Fuel, Gas).
2. **Impact and contribution** can surface as ITC (Influence Token Credits).
3. **Facts** land on a **track blockchain** and in **MusicChain logs** — not only in hidden database counters.
4. **Rules** are published (MCIPs) so third parties can verify calculations.

## Principles

| Principle | Meaning |
|-----------|---------|
| **Product before hype** | Streaming, library, live, mobile — real usage first |
| **Transparent rules** | Rates and splits documented; changes go through MCIPs |
| **Layered truth** | Events → reducer → state → UI (no mystery aggregates) |
| **Tiered fairness** | Iron base rates + membership tier multipliers (documented) |
| **Security by separation** | Public specs; private enforcement and secrets |

## Success (3–5 years)

- Creators can **trace** how a listen became rewards on-chain and in logs.
- Developers can build on **documented APIs** and SDKs.
- Community proposes improvements via **MCIPs**, not Discord rumors.
- MusicChain i3 is credible because **the math is inspectable**, not because we said so.

## Non-goals

- Replacing sovereign currencies or promising investment returns.
- Publishing exploit-friendly anti-cheat or admin internals prematurely.
- An 80-page whitepaper nobody reads — we prefer **living specs** + examples.
